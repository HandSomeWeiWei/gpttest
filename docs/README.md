# 配送平台開發文件

Codex 或開發人員請依下列順序閱讀：

1. [配送平台設計規格](配送平台設計規格.md)：系統範圍、流程、狀態、驗收與實作原則。
2. [待確認事項](待確認事項.md)：目前不可自行假設的契約與業務問題。
3. [API 規格原文轉錄](API規格原文轉錄.md)：由 Word 轉出的完整欄位、範例、SQL 與處理邏輯。
4. `../source/系統串接流程-先列印託運單版本.drawio`：原始跨系統流程。

原始檔保留在 `source/`，`docs/` 是供檢索、開發與版本審查使用的文字規格。修改原始需求後，應同步更新 `docs/`，並在 Pull Request 說明受影響的 API、狀態及測試。
